SJDC Makerspace — Implementation Plan (Units 1–9)

Target completion: August 2026

Owners:

  • Kathy — Project Manager (PM)
  • Martha — Curriculum Lead
  • Eric — Maintainer (equipment/space)

Primary constraint:

  • Makerspace includes multiple machines; maximum occupancy 30; operations require instructional support staff for safe use.

1) Plan Outputs (What “Done” Looks Like)

  • A complete hierarchical plan (this document) with checklists
  • Units 1–9 each have: lesson materials + safety briefing + setup/shutdown + assessment/checkoff + pilot + revisions
  • Safety and supervision model defined and in use (aligned to max occupancy 30)
  • Equipment inventory and maintenance system operating (owners + schedules + logs)
  • A timeline and status tracker that supports finishing by August 2026

2) Governance + Roles

  • Confirm decision/approval path (who approves: safety rules, curriculum, space changes, purchases)
  • Confirm role responsibilities
    • Kathy (PM): timeline, meeting cadence, risk log, approvals coordination, tracking
    • Martha (Curriculum): unit design, lesson materials, assessments, training content
    • Eric (Maintainer): equipment readiness, installation needs, maintenance plan, inspections
  • Set meeting cadence
    • Weekly: execution check-in (30–45 minutes)
    • Monthly: milestone review (status, risks, staffing, equipment, incidents)
  • Create documentation structure (single shared location + naming conventions)

3) Space, Capacity (Max 30), and Supervision Model

  • Define operating modes
    • Scheduled class
    • Open lab (supervised)
    • Project work time (supervised)
  • Define supervision requirements by activity category (low/medium/high risk)
  • Define staff coverage needs for each operating mode
  • Define zoning (optional) and post max counts per zone
  • Define “Stop Work” rule (any staff can pause activity for safety)

4) Safety + Procedures (Baseline for All Units)

  • Draft Makerspace Safety Handbook
  • Post signage
    • PPE requirements
    • Machine rules
    • Emergency procedures
  • Define PPE list (baseline + per-machine)
  • Define onboarding requirement (no machine use until safety onboarding completed)
  • Define incident reporting process + incident log
  • Define out-of-service procedure (tag + lockout where applicable)
  • Emergency readiness
    • First aid kit stocked and accessible
    • Fire extinguisher locations verified
    • Clear paths/egress kept unobstructed

5) Equipment Readiness + Maintenance System

  • Inventory all machines/tools
    • Name, model, serial, condition, location, power/vent needs
  • Identify installation and safety requirements (power, ventilation, dust collection, guarding)
  • Create a standard “Machine Readiness Checklist” (template)
  • Create preventative maintenance schedule
    • Owner
    • Frequency
    • Checklist
    • Log location
  • Create consumables list with minimum stock levels + reorder triggers
  • Create machine sign-off list (who is trained/cleared)

6) Curriculum Framework (Applies to Units 1–9)

  • Define learner audience(s) and prerequisites
  • Define unit structure
    • Safety briefing
    • Demo
    • Guided practice
    • Independent practice/project
    • Skills check / sign-off
  • Define assessment standards
    • Safety behaviors
    • Tool/machine competency
    • Project quality rubric (as applicable)
  • Define documentation for each unit (unit packet)
    • Instructor guide
    • Student handout
    • Setup/shutdown checklists
    • Assessment/checkoff sheet
